# Glossary

| Term | Definition |
|------|------------|
| **Wowasi_ya** | Lakota word for "assistant" (pronounced Wo-wa-shi-ya). The name of this project. |
| **Ad Hoc Agent** | A dynamically generated research agent tailored to a specific project's domain and context. |
| **Agent Strategy** | The ruleset that determines which agents to generate based on project analysis. |
| **Claude API** | Anthropic's API for accessing Claude AI models. |
| **Data Sovereignty** | The principle that data is subject to the laws and governance of the nation/entity where it originates. |
| **Domain Analysis** | The process of identifying which subject areas (health, finance, agriculture, etc.) a project involves. |
| **Obsidian** | A markdown-based knowledge management application used for documentation. |
| **Phase 0** | The agent discovery phase where project descriptions are analyzed and ad hoc agents are generated. |
| **PHI** | Protected Health Information - health data protected under HIPAA. |
| **PII** | Personally Identifiable Information - data that can identify an individual. |
| **Privacy Layer** | Application logic that sanitizes/anonymizes sensitive data before sending to external APIs. |
| **Proxmox** | Open-source virtualization platform for running VMs and containers. |
| **Starter Kit** | The standardized 15-document template structure for project documentation. |
| **Tribal IRB** | Tribal Institutional Review Board - governs research involving tribal communities. |
| **ZDR** | Zero Data Retention - API configuration where provider does not retain request/response data. |
