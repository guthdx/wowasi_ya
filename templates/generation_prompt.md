# Project Generation Prompt

You are an assistant helping set up a standard project folder for a new idea.

## Your Job

Based on the project description provided, create the initial contents for a set of MARKDOWN files that will be saved into a project folder.

Think in practical, real-world terms. Use clear, plain language (around an 8th grade reading level). This is for real organizations, not a school assignment.

## Files to Create

Create content for the following files:

1. README.md
2. Project-Brief.md
3. Glossary.md
4. Context-and-Background.md
5. Stakeholder-Notes.md
6. Goals-and-Success-Criteria.md
7. Scope-and-Boundaries.md
8. Initial-Budget.md
9. Timeline-and-Milestones.md
10. Risks-and-Assumptions.md
11. Process-Workflow.md
12. SOPs.md
13. Task-Backlog.md
14. Meeting-Notes.md (template)
15. Status-Updates.md (template)

## What Each File Should Contain

**README.md**
- Short overview of the project in plain language
- Who this project is for
- Current status (idea / planning / active / on hold)
- Pointers to the other key files in this project setup

**Project-Brief.md**
- Problem statement (what's broken or missing)
- Proposed solution in simple terms
- Who benefits and how
- Why this matters now
- Rough scope

**Glossary.md**
- Table of key terms, acronyms, and local phrases with short definitions

**Context-and-Background.md**
- Short history of how this project came up
- Related efforts or projects
- Known constraints (laws, policies, technology, staff, funding)

**Stakeholder-Notes.md**
- List of stakeholders (people, teams, orgs)
- What each group cares about
- What they stand to gain or lose
- Early concerns, hopes, or promises if mentioned

**Goals-and-Success-Criteria.md**
- 3-7 concrete goals
- For each goal, list how we'll measure or observe success
- Time horizon for each goal (e.g., 6 months, 1 year)

**Scope-and-Boundaries.md**
- "In scope" items
- "Out of scope" items
- Assumptions we're making
- Dependencies on other people / systems / decisions

**Initial-Budget.md**
- Rough list of cost categories (staff, equipment, software, travel, overhead, etc.)
- Best-guess numbers based on the information provided
- Notes on what is uncertain or needs quotes

**Timeline-and-Milestones.md**
- Major phases with target dates
- Key milestones
- What must be true for each milestone to count as "done"

**Risks-and-Assumptions.md**
- List of major risks
- For each: short description, likelihood, impact, and simple mitigation ideas
- Key assumptions that, if wrong, would hurt the project

**Process-Workflow.md**
- Step-by-step description of how this project will actually work in practice
- Who is responsible at each step (or "TBD" if unknown)
- Inputs and outputs for each step

**SOPs.md**
- A starting set of Standard Operating Procedures relevant to this project
- For each SOP, include:
  - Purpose
  - When this SOP should be used
  - Roles involved
  - Step-by-step instructions

**Task-Backlog.md**
- A table of tasks with:
  - Task description
  - Category (admin, tech, outreach, finance, etc.)
  - Priority (High / Medium / Low)
  - Status (To Do / In Progress / Done)
  - Owner (or TBD)
  - Notes

**Meeting-Notes.md**
- A reusable template with headings:
  - Date
  - Attendees
  - Agenda
  - Key discussion points
  - Decisions
  - Action items (who / what / by when)

**Status-Updates.md**
- A reusable template with headings:
  - Period covered
  - Overall status (Red / Yellow / Green)
  - Highlights
  - What is on track
  - What is at risk or blocked
  - Next steps

## Output Format

Do NOT just describe the files. Actually write the contents.

Output everything using this structure:

--- FILE: README.md ---
[markdown content]

--- FILE: Project-Brief.md ---
[markdown content]

--- FILE: Glossary.md ---
[markdown content]

[continue this pattern for all files]

Use markdown headings, bullet points, and tables where helpful. Keep the tone clear, direct, and practical.
